# ShineHeKnowledge v1.7.0 Release Notes

**Verified Hybrid**：原始文档为证据底座，已验证 Canonical Wiki 为受控增强层，维护中心负责保护与审阅。

## 亮点

1. **三档模式**：`verified`（默认）/ `authoring` / `evidence_only`（兼容 `wiki_first`/`legacy`）
2. **Serving Gate**：仅 Active + 可解析 Evidence 的 Claim 进入回答
3. **统一检索**：SearchService 内 Raw ∥ Claim → RRF 融合，Wiki 失败自动 Raw
4. **回答契约**：`answer_mode`、冲突披露、Claim+Evidence 引用、`read` 扩展
5. **维护中心**：PolicyEngine R0–R4、Jobs/Reviews/Health、API/CLI/Web
6. **MCP 边界**：`write_policy=disabled` 隐藏写工具；Authoring 显式开启
7. **Hybrid Eval**：≥150 条三路对照，门禁全绿

## 升级

见 `docs/migration/v1.6-to-v1.7-verified-hybrid.md`。

**不会自动改写**现有 `config.yaml`。建议：

```yaml
knowledge_workflow:
  mode: verified   # 或保留 wiki_first（运行时=authoring）
wiki:
  authoring_enabled: false
  auto_publish: false
mcp:
  write_policy: disabled
rag:
  verified_knowledge:
    enabled: true
```
