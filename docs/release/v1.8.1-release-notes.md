# ShineHeKnowledge v1.8.1 — Maintainability Phase 1

## Highlights

- **Request-scoped search output:** `SearchService.execute()` returns a frozen `SearchExecution` carrying `results`, `trace`, `disclose_claims`, `conflicts`, `fallbacks`, and `warnings` for a single request.
- **No shared last-request state:** removed `last_search_trace`, `last_disclose_claims`, and `get_disclose_claim_rows` from production and eval paths.
- **VerifiedAnswer migration:** `ask` consumes `execute()` only; concurrent requests cannot cross-contaminate via service instance fields.
- **Behavior contracts frozen:** Search/Ask snapshot tests plus Wiki Serving invariants (WIKI-001…010).
- **Concurrent isolation proven:** 50 concurrent `execute()` calls × 20 rounds with zero cross-request bleed.

## Compatibility

- `SearchService.search()` remains the public list-returning API (`list(execute().results)`).
- MCP tool envelopes and DB schema are unchanged.
- Retrieval ranking, RRF, rerank, citation, and Wiki Serving Gate algorithms are unchanged.
- Built on the v1.8.0 Verified Hybrid convergence baseline.
