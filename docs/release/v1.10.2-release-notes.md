# ShineHeKnowledge v1.10.2 Release Notes

> **发布日期：** 2026-07-14  
> **基线：** v1.10.1  
> **执行依据：** `docs/superpowers/specs/06-one-shot-final-closure-spec.md`

v1.10.2 是 v1.x 可维护性专项的**一次性最终收尾**。它处理 v1.10.1 竣工复核后剩余的明确问题，并正式冻结 v1.x 架构治理。

这是一次治理与一致性补丁：**无公开 API/MCP 契约变更，无检索/Wiki 语义变更，无数据库 Schema 变更，无 Alembic Revision。**

## 改动

### FIX-1：MCP 能力自描述准确

`kb_capabilities()` 此前对 `hidden_by_policy` 返回固定空数组，尽管 `RegistrationState.hidden_by_policy` 已由 `bootstrap()` 填充真实值。现已接通真实值并排序，bootstrap 未运行时回退 `[]`。

- `src/mcp/tools/retrieval.py`：`hidden_by_policy = sorted(state.hidden_by_policy) if state else []`
- 测试：`tests/mcp/test_kb_capabilities_policy_state.py`（5 例：无隐藏 / 有隐藏 / 排序保证 / 未 bootstrap / list 类型）

### FIX-2：Wiki 契约进入独立 Contract Gate

Wiki 服务不变量（`tests/test_wiki_serving_contract.py`）此前仅由全量 Test Job 间接覆盖。现纳入独立 Contract Gate，与 Search/Ask/MCP 四契约并列显式运行；gate 名称与 step 名称均含 Wiki。

- `.github/workflows/ci.yml`：`contract-gate` 显式运行四契约
- 静态测试：`tests/architecture/test_ci_workflow_contracts.py`（断言四契约、`ruff check .`、debt strict 在册，且无 `continue-on-error: true` / `|| true` 绕过）

### FIX-3：Ruff 覆盖整个仓库

CI lint 从 `ruff check src tests evals tools scripts` 改为 `ruff check .`，覆盖 Alembic Revision 与整个仓库。

### FIX-4：前后端版本元数据统一 1.10.2

版本元数据此前漂移严重：`src/version.py`=1.10.1，`client/package.json`=1.9.0，`client/package-lock.json` 根包=1.4.0。统一为 1.10.2，并新增版本一致性测试防止再次漂移。

- `src/version.py`、`client/package.json`、`client/package-lock.json`、`README.md`、`README_zh.md`、`PROGRESS.md`、`docs/README.md`
- 测试：`tests/architecture/test_version_consistency.py`（6 例：src / README badge / README_zh badge / package.json / lockfile 根包两处一致性）

### FIX-5：发布与远端 CI 闭环

全量验收通过后，提交、推送、确认远端 CI 全绿，创建 `v1.10.2` Tag 与 GitHub Release。

## v1.x 架构冻结

v1.10.2 发布后，v1.x 可维护性专项正式关闭。剩余兼容层与架构优化登记于 `docs/architecture/v1-maintainability-freeze.md`，转入 v2.0。后续工作默认回归产品功能开发；任何进一步架构优化必须进入 v2.0 规划，不再作为 v1.x 竣工条件。
