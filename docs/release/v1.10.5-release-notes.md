v1.10.5 修复生产试点评估可信度与 Provider 运行链路中的三项基础机制问题：建立可审计 Ground Truth 审核/冻结流程，修复 Routing Harness 参数与任务结果真实性，并把可终止进程隔离接入正式 LLM、Embedding 与 Reranker 路径。

**无数据库 Schema 变更，无 Alembic Revision。正式 `data/kb.db` 大小与 SHA256 均未变化。**

> **验收结论：** 本版本不声称达到生产试点门槛。196 条候选尚未完成真实双人审核，`frozen/` 保持为空，因此不能进入独立全量验收。

## Ground Truth 审核与冻结

- 规则生成数据只输出 `rule_assisted_candidate` / `pending`，不再冒充 `human`
- `candidates/`、`reviewed/`、`frozen/` 三层数据角色分离
- 新增人工审核 CLI：标题与正文检查、expected/acceptable/forbidden/irrelevant 决策、独立二审与争议裁决
- Answer Citation 要求真实 block ID、原文 quote 与对应 knowledge ID
- freeze 严格校验审核员、ISO 8601 时间、证据、快照、争议状态与正文 quote
- 正式评估与 MCP Harness 只读 `frozen/`
- 当前真实状态：196 candidates / 0 reviewed / 0 frozen

## Routing Harness

- 以 `deepcopy(recommended_arguments)` 原样执行 Agent 推荐参数
- 保存 raw/executed arguments、精确一致性、契约检查与原始 route/exec 响应
- 统一区分 empty、non_empty、no_answer、graph/structured result、validation/provider/MCP/transport error、timeout 与 cancelled
- 空结果、timeout、validation error、transport error 均不计为任务完成
- 支持按 Agent 明示映射执行 search → graph_traverse 多步推荐 flow
- 真实 10 条 stdio candidate smoke：参数保真 10/10；7 validation error、2 non_empty、1 empty，未掩盖失败

## 正式 Provider 隔离

- 非流式 LLM generation：`process`
- Embedding：`process`
- API Reranker：`process`
- 本地 CrossEncoder Reranker：`process`
- LLM fallback Reranker 通过中心 LLM 服务获得相同隔离
- LLM streaming 显式使用 `thread_cooperative` 与请求 timeout
- timeout 后 terminate/join，必要时 kill/join；关闭 queue；worker 数量受限
- ProviderRequest 不携带 Container、DB/SQLite 连接、HTTP session、模型实例或明文密钥
- `kb_health_check` 新增无秘密的 `provider_isolation` 诊断

真实 Provider smoke：LLM 正常 3/3、人工短 timeout 2/2 且 worker 已终止；Embedding 正常 3/3；Reranker 正常 3/3。
