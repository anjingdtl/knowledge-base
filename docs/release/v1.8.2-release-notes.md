# ShineHeKnowledge v1.8.2 — Maintainability Phase 2

## Highlights

- **Retrieval orchestration package:** `src/retrieval/` introduces `RawRetriever`, `VerifiedProvider`, `EvidenceOnlyPolicy`, `VerifiedPolicy`, and `RetrievalOrchestrator`.
- **SearchService facade:** `execute()` routes through the orchestrator; public `search()` / `SearchExecution` semantics unchanged.
- **Safe cutover modes:** `retrieval.orchestrator` supports `legacy` (default), `shadow`, and `unified`.
- **Wiki serving isolation:** claim serving goes through `VerifiedProvider` + existing Serving Gate; wiki failures never block raw evidence.
- **Shadow comparator:** compares source IDs, claims, conflicts, fallbacks, and citations without logging full document text.

## Compatibility

- Default remains **legacy** primary pipeline (full rollback by config).
- MCP tools, AppContainer wiring, and DB schema are unchanged.
- RRF, rerank, citation, and gate **algorithms** are unchanged (adapter extraction only).
- Built on v1.8.1 Phase-1 contracts (`SearchExecution`, no `last_*` state).
