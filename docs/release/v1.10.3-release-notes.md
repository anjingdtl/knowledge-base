v1.10.3 是 **MCP 最终收口与本地真实 MCP 验证** 发布。关闭 timeout / Graph / Structured / no-answer 等剩余代码风险，并用真实 `stdio` 与 `streamable-http` 完成并发、长稳与 107 Golden 验收。

**无数据库 Schema 变更，无 Alembic Revision。** 公开工具名与主要 MCP 配置档语义保持兼容。

## 改动摘要

### Timeout 真取消与诚实语义

- 新增 `src/services/deadline.py`：统一 deadline、cancel event、`DeadlineTimeout`
- 仅在工作真正结束后标记 `cancelled=true`；否则 `background_work_may_continue=true`
- 超时后不永久占用请求 slot；cooperative 连续 50 次 timeout 线程可控
- Python 3.10 兼容：`concurrent.futures.TimeoutError` 与 builtins `TimeoutError` 均映射为 ask timeout 信封
- `route_query` LLM 路由注入 `rag.route_llm_timeout`

### Graph 硬上限分页

- `paginate_graph_result`：禁止 `next_offset == offset` 自循环
- 到达 `max_graph_nodes` 返回 `hard_limit_reached` 与 `next_offset=null`
- `graph_traverse` / `execute_query(type=graph)` 统一行为

### Structured Query 分页

- 多取一条（`effective_limit + 1`）判断 `has_more`
- `meta.limit` = effective_limit；`total_estimate` 不伪精确
- 稳定次级排序 `id ASC`

### No-answer / FTS 门禁

- `src/services/relevance_gate.py`：语义/FTS/标题/短语/数字单位统一评分
- 当前信息类查询（今天/股价/实时…）本地库无实时源时 no-answer
- Ask 弱证据阻断编造答案

### 工程与验证

- 真实 MCP harness：`scripts/final_closure_mcp_harness.py`
- CI：`fix/**` 分支触发；远程 CI 全绿
- Windows 子进程 UTF-8：`mcp_launcher._run_hidden` `errors=replace`
