# v1.7 → v1.8：Verified Hybrid 收束纠偏迁移

> 状态：随 v1.8.0 交付；运行时兼容旧配置，不会自动写入用户文件。

## 先备份，再预览

```powershell
shinehe config migrate-verified-hybrid --config .\config.yaml --dry-run
```

该命令不写文件。`wiki_first` 在运行时仍等价于 `authoring`；`legacy` 仍等价于 `evidence_only`。只有在明确决定规范化配置时才执行 apply。

## Apply 与回滚

```powershell
# 仅规范旧别名，保留原运行档位
shinehe config migrate-verified-hybrid --config .\config.yaml --apply

# 明确切换为默认 Verified 档
shinehe config migrate-verified-hybrid --config .\config.yaml --apply --target-mode verified

# 使用 apply 输出的 backup_path 恢复原始字节
shinehe config migrate-verified-hybrid --config .\config.yaml --rollback .\config.yaml.verified-hybrid-<timestamp>.bak
```

apply 会在同目录创建备份并通过临时文件原子替换；未知字段保持不变。`verified` 会强制关闭 Agent 写与自动发布，`authoring` 不会自动发布。

## Claim 与 Maintenance schema

```powershell
shinehe wiki serving-validation-migration --dry-run
shinehe wiki migrate-v2 --dry-run
```

缺少当前 revision 的 review、validation 或 publish 证明的 Active Claim 不会被迁移工具伪造为可 Serving；它们必须进入 Review → Validator → 显式 Publish。Maintenance 的 Alembic 迁移仅增加控制面表和索引，不改写 Raw 或 Canonical 正文。

## 验证与降级

- 先运行 `shinehe doctor --explain-config`、`shinehe maintenance health` 和 `scripts/verified-hybrid-acceptance.ps1`。
- Wiki 或 Maintenance 异常时，`search` / `ask` 应回退 Raw；可临时设置 `knowledge_workflow.mode: evidence_only`。
- 配置回滚使用上面的 `.bak`；Canonical V2 回滚使用 `shinehe wiki migrate-v2 --rollback <timestamp>`。
