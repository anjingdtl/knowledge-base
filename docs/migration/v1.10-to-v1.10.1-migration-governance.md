# v1.10.0 тЖТ v1.10.1 Migration Governance

## Summary

v1.10.1 closes Spec 05 (final migration governance). **Alembic becomes the sole runtime schema creation & upgrade authority.** This is a governance patch: no public contract, retrieval, or Wiki semantics change, and no new database revision.

| Work package | Outcome |
|---|---|
| WP0 | Final governance baseline frozen; `tools/schema_fingerprint.py` added |
| WP1 | CI architecture gate made strict (`report_closure_debt --strict`, non-zero on debt) |
| WP2 | Migration Gate moved before `Database.open_runtime()`; read-only boot uses `file:...?mode=ro` |
| WP3 | New/empty DB created exclusively via Alembic `upgrade_to_head` (`auto_upgrade_empty`, default `true`) |
| WP4 | `allow_unstamped=false` default; `shinehe db` CLI; backup тЖТ detect тЖТ stamp тЖТ upgrade тЖТ verify; failure auto-restore |
| WP5 | Runtime no longer runs `_SCHEMA` / `_migrate()` (moved to `src/compatibility/runtime_schema_migrate.py`) |
| WP6 | Full acceptance + release (this version) |

## Config changes

| Key | Before (v1.10.0) | After (v1.10.1) |
|-----|--------|-------|
| `storage.migration_gate.allow_unstamped` | effectively allowed (warned) | **`false`** тАФ non-empty unstamped DB refuses write boot |
| `storage.migration_gate.auto_upgrade_empty` | n/a (runtime `_SCHEMA`) | **`true`** тАФ missing/empty DB тЖТ `alembic upgrade head` |
| Runtime `_SCHEMA` / `_migrate()` | executed on boot | **removed from production boot path** |

## Operator notes

1. **No schema change** vs v1.10.0 if already at Alembic head тАФ boot is idempotent.
2. A non-empty database without `alembic_version` now **refuses write boot** by default. Run `shinehe db status`, then `shinehe db migrate` (backs up, detects, stamps, upgrades, verifies).
3. Behind-head write boot is refused; use `storage.readonly: true` (or `SHINEHE_READONLY=1`) for diagnostic read-only boot.
4. `SHINEHE_SKIP_MIGRATION_GATE=1` is **emergency-only**, not for production.

## db CLI

```bash
shinehe db status                              # revision / head / unstamped / write-allowed / recommendation
shinehe db backup                              # SQLite Backup API snapshot: <db>.backup-<ts>.sqlite
shinehe db migrate                             # backup тЖТ detect тЖТ stamp тЖТ upgrade head тЖТ fingerprint тЖТ integrity
shinehe db stamp --from-version v1.9.0         # explicit; detector must match (no blind `stamp head`)
shinehe db verify                              # integrity_check / foreign_key_check / head / row-count
```
