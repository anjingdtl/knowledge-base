# v1.8 → v1.9：可维护性三期迁移摘要

> **状态：** 随 **v1.9.0** 交付。无数据库 Schema 强制迁移；配置可选。

## 版本对应

| 版本 | 内容 |
| --- | --- |
| v1.8.0 | Verified Hybrid 融合收束纠偏 |
| v1.8.1 | 可维护性一期：`SearchExecution`、契约冻结 |
| v1.8.2 | 可维护性二期：Retrieval Orchestrator |
| **v1.9.0** | 可维护性三期：Answer / MCP / Container / DB 治理 |

## 配置（可选）

```yaml
# 检索编排（二期，默认 legacy 正式路径）
retrieval:
  orchestrator: legacy   # legacy | shadow | unified

# 问答编排（三期，默认 unified；与 legacy 结构字段一致）
answer:
  orchestrator: unified  # legacy | shadow | unified
```

省略上述块时：

- 检索：按 **legacy** 主路径运行（可回滚）
- 问答：按 **unified**（`AnswerService`）运行

## 代码入口变化（兼容）

| 旧用法 | 新用法 | 说明 |
| --- | --- | --- |
| `VerifiedAnswerService.ask` | 内部委托 `AnswerService` | 对外仍可用 |
| `from src.mcp_server import mcp` | 仍可用 | 实现在 `src.mcp.server` |
| `SearchService.execute` | 不变 | 经 Orchestrator |
| 直接依赖 `Database._instance` | 弃用 | 见弃用登记 |

## 数据库

- **无** v1.9 专用 Alembic revision 强制要求
- 运行时 `Database._migrate()` **冻结**：新字段/表必须走 Alembic  
  见 `docs/architecture/database-migration-policy.md`
