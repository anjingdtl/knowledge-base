## 1. 产品变化摘要

ShineHeKnowledge 统一为：

> 本地优先、面向 AI Agent 的可验证知识检索引擎：原始文档为证据底座，Canonical Wiki V2 为受控知识增强层。

三个运行档位（同一产品，不是三个产品）：

| 模式 | 用途 | MCP 写 | Wiki 读 | Wiki Authoring |
|---|---|---|---|---|
| `verified`（**新默认**） | 日常 Agent 检索 | 默认关闭 | 已验证 Claim | 关闭 |
| `authoring` | 构建/维护 Canonical Wiki | `local_confirm` 等 | 开 | 开 |
| `evidence_only` | 降级、调试、对照评测 | 关闭 | 关 | 关 |

## 2. 配置兼容

| 旧值 | 解析为 | 行为 |
|---|---|---|
| `wiki_first` | `authoring` | **保持维护能力**，不切只读 |
| `legacy` | `evidence_only` | 仅原始检索 + 弃用提示 |
| 未配置 | `verified` | 新默认 |
| 非法值 | 启动/Doctor 报错 | 不静默猜测 |

**不会自动改写**用户 `config.yaml`。Doctor 会提示将旧值改为规范名。

### 读写分离（新字段）

```yaml
wiki:
  read_enabled: true          # 回答时是否允许读 Verified Wiki
  authoring_enabled: false    # 是否允许维护/编译
  serving:                    # Phase 2 Gate 消费
    enabled: true
    on_failure: raw_fallback
```

`wiki.canonical_v2.mode` 仍只控制 **写入迁移阶段**，不再承担“回答是否读 Wiki”职责。

## 3. 初始化

```bash
# 默认 verified：不创建 Authoring 目录，不开启写入
shinehe init --local --path D:\docs --client claude-code

# 显式 Authoring：创建 raw/wiki/schema 布局
shinehe init --mode authoring --local --path D:\knowledge

# 仅证据检索
shinehe init --mode evidence-only --local --path D:\docs
```

CLI 使用连字符 `evidence-only`；YAML 使用 `evidence_only`。

## 4. 现有 wiki_first 用户

1. **无需立刻改配置**：`mode: wiki_first` 运行时等价 `authoring`。
2. 建议在人工维护窗口改为：

```yaml
knowledge_workflow:
  mode: authoring
wiki:
  read_enabled: true
  authoring_enabled: true
  auto_publish: false
```

3. 若希望切到默认产品档：

```yaml
knowledge_workflow:
  mode: verified
wiki:
  authoring_enabled: false
  auto_compile: false
mcp:
  tool_profile: core
  write_policy: disabled
  experimental_tools_enabled: false
```

## 6. 分步迁移（生产）

1. **备份** `data/` 与 `config.yaml`
2. **升级** 到 v1.7.0 代码
3. **保持旧 mode 试跑**（`wiki_first` 仍可用）→ `shinehe doctor`
4. **验证检索**：`search` / `ask` 与 `python evals/run_hybrid_eval.py --strict`
5. **切换默认**（可选）：`mode: verified` + `write_policy: disabled`
6. **Authoring 用户**：显式 `mode: authoring` + `authoring_enabled: true` + 合适 write_policy
7. **维护**：`shinehe maintenance health`；来源变更走 index / source-event

无需破坏性数据库迁移即可使用 Serving 读路径；Canonical V2 写路径沿用既有 migration 手册。

## 7. 回滚

- **配置**：改回 `wiki_first` / `legacy` 或设 `rag.verified_knowledge.enabled: false`
- **代码**：按 Phase 回滚对应提交；Serving 读路径无强制 schema 升级
- **维护**：关闭 `maintenance.enabled` 不影响 Raw
