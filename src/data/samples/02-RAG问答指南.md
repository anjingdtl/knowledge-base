# RAG 智能问答使用指南

## 什么是 RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合了信息检索和文本生成的 AI 技术。ShineHeKnowledge 的 RAG 流程包含 6 个阶段：

## 管线阶段

### 1. 查询改写 (Query Rewrite)
当你输入一个问题时，系统会使用 LLM 生成多个版本的查询，以覆盖不同的表达方式，提高召回率。

### 2. Wiki 检索 (Wiki Retrieval)
搜索 Wiki 系统中的结构化知识，优先使用经过审核的高质量内容。

### 3. 向量搜索 (Vector Search)
使用混合搜索策略——向量搜索（语义相似度）+ 关键词搜索（精确匹配），通过 Reciprocal Rank Fusion (RRF) 算法融合结果。

### 4. 重排序 (Rerank)
使用专门的 Reranker 模型对候选结果进行精确打分，过滤掉不相关的内容。

### 5. 回答生成 (Generate)
将检索到的上下文和你的问题一起发送给 LLM，生成最终回答。

### 6. 后处理 (Postprocess)
去除重复引用，截断过长的回答，确保输出整洁。

## 使用技巧

- **具体化提问**：与其问"Python 怎么用"，不如问"如何在 Python 中读取 CSV 文件并按列筛选数据"
- **引用来源**：每个回答下方会显示引用的知识条目，你可以点击查看原始内容
- **保存到 Wiki**：如果某个回答特别有价值，可以点击"保存到 Wiki"将问答保存为知识页面
- **连续对话**：系统支持上下文对话，你可以在前一个问题的基础上继续追问
